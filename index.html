Méthodologie

Même prompt envoyé textuellement aux 4 modèles via Claude Code (sessions fraîches, contrainte de 16k tokens output). Trois ennemis distincts, HUD, particules, shadow mapping. Aucune retouche post-génération.

Meilleure expérience sur Chrome desktop (Pointer Lock + ZQSD/WASD).

Prompt utilisé 
Crée un jeu 3D complet en HTML/JavaScript single-file utilisant Three.js (CDN: https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js).

CONCEPT : "Survival Arena" - Le joueur contrôle un cube qui doit survivre dans une arène circulaire pendant que des sphères ennemies spawneent et le poursuivent. Récolter des cristaux dorés pour gagner des points.

EXIGENCES TECHNIQUES :
- Single file HTML, aucune dépendance externe sauf Three.js CDN
- Pas de localStorage/sessionStorage
- 60 FPS stable, requestAnimationFrame propre avec deltaTime
- Pointer Lock API pour la caméra FPS/TPS
- Physique simple maison (pas de lib externe) : gravité, collisions AABB+sphère

GAMEPLAY :
- Déplacement ZQSD/WASD + Espace pour sauter
- Caméra TPS qui suit le joueur avec offset configurable
- 3 types d'ennemis avec comportements distincts :
  * Chasseur (rouge) : poursuit en ligne droite
  * Embusqueur (violet) : se téléporte près du joueur toutes les 5s
  * Sentinelle (orange) : patrouille un cercle, tire des projectiles
- Spawn progressif (1 ennemi/10s, max 15)
- Cristaux dorés qui spawneent aléatoirement, +10 points
- Vie du joueur : 100 HP, dégâts au contact, régen lente hors combat
- Game over avec écran de fin et score

QUALITÉ VISUELLE :
- Lumière directionnelle + ambient + shadow mapping
- Sol avec texture procédurale (canvas 2D générée)
- Particules au ramassage de cristal et à la mort d'ennemi
- HUD overlay HTML : HP bar, score, vague actuelle, ennemis restants
- Skybox gradient ou couleur unie atmosphérique
- Brouillard exponentiel pour la profondeur

STRUCTURE DU CODE :
- Architecture en classes ES6 : Game, Player, Enemy (+ sous-classes), Crystal, ParticleSystem, InputManager, HUD
- Game loop avec états : MENU, PLAYING, GAME_OVER
- Code commenté en français, sections clairement séparées

LIVRABLE : un seul fichier .html prêt à ouvrir dans Chrome, fonctionnel au premier essai, sans erreur console.

Reproductible : clone le repo, copie ce prompt dans Claude Code, compare tes résultats.
